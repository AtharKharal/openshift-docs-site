Incident Response Plan

This document details the steps to be taken in the event of a security incident.

Response Phases

  1. Detection – Monitor logs and alerts.

  2. Analysis – Triage and classify severity.

  3. Containment – Limit the blast radius.

  4. Eradication – Remove root cause.

  5. Recovery – Restore services safely.

  6. Post-Incident Review – Document lessons learned.

Severity Classification

Level Response Time Description

Critical

30 min

Data breach or system compromise.

High

2 hours

Malware detected, but no data loss.

Medium

1 day

Suspicious activity requiring review.

Low

3 days

Informational alerts or false positives.

Contact the Security Officer immediately after detecting a Critical or High-severity incident.
